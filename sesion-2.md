# HTTP y HTTPS.

- HTTPS es una versión más segura porque se cifra la información.
- El backend utiliza HTTPS porque se comunica información que debe ser segura.
- Existen handlers que captura información de APIs que no están protegidas.
- Crear un aplicativo es fácil, una de las cosas más importante es crearlo de manera segura.
  - ![alt text](image-1.png)

## Métodos HTTP.

- Los más usados en apps:
  1. GET
  2. POST.
  3. PUT/PATCH.
  4. DELETE.

## JSON.

- JavaScript Object Notation.
- Texto plano con estructura clave - valor.
- Compatible con casi todos los lenguajes (C#, Java, TypeScript, Python, etc).
- Ideal para API REST.
- SOAP ya no se usa; utiliza XML.

## Resumen del flujo.

1. Controller.
2. Endpoint.
3. La API siempre retorna JSON.

# Ciclo de vida de una app.

1. Análisis.
   - ¿Qué problemas resolvemos?
   - Entender alcance.
   - Saberlo capitalizar.
2. Diseño.
   - ¿Cómo lo vamos a resolver?
3. Desarrollo.
   - Construcción de la app.
4. Pruebas.
   - Verificar que funcione bien.
5. Publicación.
   - Subir a las tiendas o entregar al cliente.
6. Mantenimiento.
   - Mejorar, actualizar y corregir con el tiempo.
